🛡️ What is a DNS Leak?

A DNS leak occurs when your DNS requests are sent to your ISP's servers instead of your VPN's DNS servers, potentially exposing your browsing activity even when using a VPN.

How to Fix DNS Leaks:

  • Use VPN DNS Servers: Configure your VPN to use its own DNS servers
  • Enable DNS Leak Protection: Most VPNs have this built-in
  • Use Encrypted DNS: Enable DNS-over-HTTPS (DoH) or DNS-over-TLS (DoT)
  • Change DNS Settings: Use Cloudflare (1.1.1.1) or Quad9 (9.9.9.9)
  • Disable IPv6: Some VPNs don't support IPv6, causing leaks

🔐 Recommended Privacy DNS Providers

  • Cloudflare (1.1.1.1): Fast, privacy-focused, supports DoH/DoT
  • Quad9 (9.9.9.9): Blocks malicious domains, privacy-focused
  • AdGuard DNS (94.140.14.14): Blocks ads and trackers
  • OpenDNS (208.67.222.222): Additional security features

🛡️ What is a WebRTC Leak?

WebRTC is a browser feature enabling video/audio chat. It can reveal your real IP address even when using a VPN by operating independently of the VPN tunnel.

How WebRTC Leaks Occur:

  • STUN Requests: WebRTC uses STUN servers to discover your public IP
  • Local Network Discovery: Can expose your local (private) IP addresses
  • VPN Bypass: Operates independently of VPN tunnel
  • Browser Default: WebRTC is enabled by default in most browsers

🔧 How to Prevent WebRTC Leaks

  • Firefox: Set media.peerconnection.enabled to false in about:config
  • Chrome/Edge: Install "WebRTC Leak Prevent" extension, set to "Disable non-proxied UDP"
  • Brave: Settings → Privacy → WebRTC IP handling → "Disable non-proxied UDP"
  • Safari: Develop → Experimental Features → Disable WebRTC
  • Any browser: Use uBlock Origin with advanced settings enabled
